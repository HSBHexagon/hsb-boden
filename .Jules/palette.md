## 2026-07-18 - [Add explicit required indicators and focus states to LeadForm]
**Learning:** Native browser `required` attributes and default focus states are often insufficient for good accessibility and UX, especially in complex grids. Explicit visual indicators (like a red asterisk with `aria-hidden="true"`) prevent user frustration before form submission. Additionally, adding explicit `focus-visible` classes ensures keyboard navigation is clearly visible across different browsers and operating systems where native outlines might be overridden or stripped by CSS resets. It is important to wrap the label text and indicator in a `span` container to prevent CSS grid layout breaks.
**Action:** Always verify keyboard navigation visually and ensure native validation has clear corresponding visual indicators. Use `focus-visible` over plain `focus` to reduce visual noise for mouse users while supporting keyboard users.
