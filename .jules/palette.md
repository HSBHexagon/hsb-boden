## 2026-07-17 - Form Label Accessibility and Focus Visibility
**Learning:** Native `required` indicators lack visual cues and standard Tailwind configurations don't always provide sufficient focus states across diverse interactive elements (inputs, buttons, checkboxes, links). Complex label text formats (like wrapping `<span class="text-hsb-red" aria-hidden="true">*</span>` alongside text in a parent `<span>`) are necessary when dealing with CSS grid layouts to ensure label elements stay grouped properly on the same line.
**Action:** Applied explicit explicit `focus-visible` ring classes to all interactive form elements and `CookieConsent` components, and systematically added ARIA-hidden asterisk markers to required inputs for visual clarity for sighted users while preserving structural layout via nested `<span>`.
